1. Latar Belakang Shopee

Shopee merupakan ekosistem e-commerce masif yang membutuhkan manajemen data yang sangat kompleks. Pendekatan pemrograman prosedural tidak lagi efisien untuk mengelola jutaan interaksi produk dan user. Paradigma OOP diterapkan untuk menciptakan sistem yang modular, aman, dan mudah dikembangkan di masa depan.

2. Masalah yang Sering Dihadapi

  • Sinkronisasi Stok: Data stok tidak sinkron secara real-time saat terjadi transaksi massal (misal: flash sale).
  • Kompleksitas Skalabilitas: Penambahan kategori produk baru sering menyebabkan sistem menjadi tidak stabil.
  • Integritas Data: Risiko manipulasi data transaksi jika akses atribut tidak dibatasi (kurangnya enkapsulasi).

3. Mengapa OOP?

OOP mempermudah pengembangan dengan konsep Encapsulation (keamanan data), Inheritance (penggunaan kembali kode), dan Modularity (pemecahan sistem menjadi bagian-bagian kecil yang mudah dipelihara).

Source Code 1: product.dart (Abstraksi)

ini berfungsi sebagai kontrak sistem yang memastikan standarisasi seluruh produk. Sebagai Cetakan Umum: Kata kunci abstract menegaskan bahwa Product adalah kategori umum. Kita tidak bisa membuat objek produk tanpa jenis yang spesifik, sehingga data di sistem selalu teratur. Standarisasi Atribut: Setiap produk wajib memiliki name dan price. Ini memudahkan modul lain (seperti Keranjang) mengolah data produk dengan format yang seragam. Kewajiban Perilaku: Metode displayInfo() memaksa setiap kelas turunan untuk memiliki fungsi tampilan sendiri. Jika tidak, sistem akan menolak (error), sehingga konsistensi perilaku objek tetap terjaga.

abstract class Product {
  String name;
  double price;
  Product(this.name, this.price);
  void displayInfo();
}

Source Code 2: electronic.dart (Inheritance)

Menerapkan pewarisan untuk memperluas fungsi Product. Kelas ini mewarisi atribut dari Product dan memberikan implementasi spesifik pada metode displayInfo(), sehingga kita tidak perlu menulis ulang kode dasar. 
import 'product.dart';
class Electronic extends Product {
  Electronic(String name, double price) : super(name, price);
  @override
  void displayInfo() => print("Elektronik: $name - Rp$price");
}

Source Code 3: cart.dart (Composition)

Mengelola kumpulan produk dalam sebuah keranjang. Kelas ini menggunakan list untuk menyimpan berbagai objek Product dan menyediakan metode terpusat untuk menghitung total harga (getTotal), sehingga logika transaksi tetap konsisten. 
import 'product.dart';
class Cart {
  List items = [];
  void addItem(Product p) => items.add(p);
  double getTotal() => items.fold(0, (s, i) => s + i.price);
}

Source Code 4: user.dart (Encapsulation)

Source Code 4: user.dart (Encapsulation)

privasi data. Variabel _name yang bersifat private tidak bisa diakses langsung dari luar kelas, melainkan harus melalui metode yang disediakan. Ini mencegah manipulasi data oleh pihak yang tidak berwenang.

import 'cart.dart';
class User {
  String _name;
  Cart cart = Cart();
  User(this._name);
  String get name => _name;
}

Source Code 5: main.dart (Eksekusi)

Source Code 5: main.dart (Eksekusi)

Pusat kendali (integrasi). Di sini seluruh objek (User, Cart, Product) disatukan untuk menjalankan alur belanja yang nyata, mulai dari inisialisasi user hingga perhitungan akhir total pembayaran.

import 'user.dart';
import 'electronic.dart';

void main() {
  var user = User("Serlin & Jeinnet");
  user.cart.addItem(Electronic("Smartphone", 5000000));
  print("Total Pembayaran: Rp${user.cart.getTotal()}");
}
